# ArtPage → 모노페이지 (Monopage) 리브랜딩

- 날짜: 2026-03-31
- 에이전트: [TCC]

## 세션 개요

도메인 `monopage.kr` 구매를 계기로 artpage 프로젝트를 모노페이지로 전면 리브랜딩.
패키지명, 브랜드 텍스트, 도메인, 세션스토리지 키, NCP 경로 등 전체 교체.

## 주요 변경사항

### 브랜드 교체
- `ArtPage` → `모노페이지`
- `ARTPAGE ADMIN` → `MONOPAGE ADMIN`
- 로고 문자 `A` → `M`

### 패키지
- `@vibers/artpage` → `@vibers/monopage`

### 도메인
- 플랫폼: `artpage.vibers.co.kr` → `monopage.kr`
- 테넌트1: `arthyun.co.kr` (기존 유지) + `arthyun.monopage.kr` (신규 추가)
- 테넌트2: `artway.vibers.co.kr` → `art-way.monopage.kr`

### 미들웨어 (middleware.ts)
- `*.monopage.kr` 서브도메인 자동 라우팅 추가
- `arthyun.monopage.kr` → `/arthyun/...`
- `art-way.monopage.kr` → `/art-way/...`

### 세션스토리지 키
- `artpage_create_*` → `monopage_create_*`

### NCP 스토리지 경로
- `artpage/create` → `monopage/create`
- default category: `artpage` → `monopage`

## 추가 작업 (같은 날 이어서)

### art-way 아카이브 페이지 크래시 수정
- 원인1: Supabase 데이터 삭제로 전시 데이터 없음 → try/catch + 빈 상태 UI 추가
- 원인2: Firebase env vars 미설정 → `auth`가 undefined → `onAuthStateChanged` 크래시
- 수정: `AdminButtons.tsx`에 `if (!auth) return;` 방어 처리 추가
- 수정: Vercel monopage 프로젝트에 `NEXT_PUBLIC_FIREBASE_*` 6개 환경변수 추가
- 결과: `art-way.monopage.kr/archive` 정상 동작 확인

### artway 별칭 추가
- `artway.monopage.kr`도 `/art-way`로 라우팅되도록 미들웨어에 추가

### art-way 독립 Vercel/Git 프로젝트 삭제
- 데이터는 NCP에 있으므로 독립 프로젝트 삭제해도 무방
- 현재 monopage 하위로 완전 통합

- `www` → CNAME `cname.vercel-dns.com`

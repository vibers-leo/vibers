# 플랫폼과 데모의 구조적 분리 및 에디터 구축

**(부제: 2025-12-28, SaaS 아키텍처 전환과 위지윅 에디터 구현)**

## 발단: 단일 사이트에서 플랫폼으로의 확장

레오는 기존에 구축한 '북촌 아트 스페이스' 웹사이트를 단일 갤러리 페이지가 아닌, 다수의 갤러리가 입점할 수 있는 'ArtPage'라는 SaaS 플랫폼으로 확장하고자 했다. 그는 각 갤러리 관리자가 자신의 페이지를 직접 수정할 수 있는 블록 기반의 에디터 기능을 요청했으며, 동시에 메인 플랫폼 페이지와 개별 갤러리 템플릿(데모)이 명확히 구분되기를 원했다.

## 전개: Tiptap 에디터 도입과 라우팅 그룹 설계

안티그래비티는 관리자가 직관적으로 콘텐츠를 편집할 수 있도록 Tiptap 기반의 위지윅(WYSIWYG) 에디터 도입을 제안했다. Tiptap 라이브러리와 관련 의존성 패키지를 설치한 후, 관리자 대시보드와 페이지 편집 기능을 구현하여 콘텐츠 관리 시스템(CMS)의 기초를 다졌다.

이어 레오는 `bstage.in`을 레퍼런스로 제시하며, 메인 페이지는 플랫폼의 기능을 소개하는 쇼케이스로, 기존 작업물은 실제 작동하는 데모 사이트로 분리할 것을 지시했다. 이에 안티그래비티는 Next.js의 Route Groups 기능을 활용하여 `(platform)`과 `(tenant)`로 디렉토리 구조를 물리적으로 분리하는 작업을 시작했다.

## 위기: 경로 변경에 따른 네비게이션 불일치

물리적인 파일 이동과 레이아웃 분리는 순조롭게 진행되었으나, 기존 '북촌' 템플릿에서 사용하던 공통 컴포넌트인 `Header.tsx`와 `MobileMenu.tsx`에서 문제가 발생했다. 해당 컴포넌트들은 루트 경로(`/`)를 기준으로 작성되어 있었기 때문에, `/demo` 하위로 이동한 데모 사이트 내에서 링크가 올바르게 작동하지 않았다. 안티그래비티가 이를 수정하기 위해 부분 코드 교체(Search & Replace)를 시도했으나, 파일 내용의 불일치로 인해 코드 적용에 실패하는 오류가 발생했다.

## 절정: 안전한 전면 재작성과 레이아웃 이원화

안티그래비티는 부분 수정 방식 대신, `Header.tsx`와 `MobileMenu.tsx`의 전체 코드를 `/demo` 경로에 맞춰 재작성하는 방식으로 선회하여 문제를 해결했다. 이 과정에서 `layout.tsx`를 최상위, 플랫폼용, 테넌트용으로 계층화하여 각 영역이 독립적인 헤더와 푸터를 가지도록 설계했다. 메인 루트(`/`)에 접속하면 SaaS 소개 페이지가, `/demo`에 접속하면 갤러리 템플릿이 렌더링되도록 라우팅 구조를 확정 지었다.

## 결말: 이원화된 서비스 구조 완성

작업 결과, ArtPage 프로젝트는 '서비스를 판매하는 플랫폼'과 '서비스가 적용된 결과물'이 공존하는 구조를 갖추게 되었다. 레오는 관리자 에디터와 분리된 데모 사이트를 확인하며 만족감을 표했다. 이번 협업을 통해 단일 페이지 애플리케이션을 다중 목적의 플랫폼으로 확장할 때, 공통 컴포넌트의 경로 의존성을 어떻게 관리해야 하는지에 대한 기술적 아키텍처가 정립되었다.

### 🛠 기술적 각주 (Technical Summary)

**수정된 파일 (Modified Files):**

- `src/app/(platform)/...`: 메인 플랫폼 라우트 그룹 및 관련 페이지 신설
- `src/app/(tenant)/...`: 데모 사이트 라우트 그룹 및 기존 갤러리 페이지 이동
- `src/components/Header.tsx`, `src/components/MobileMenu.tsx`: `/demo` 하위 경로 지원을 위한 링크 수정
- `src/components/editor/BlockEditor.tsx`: Tiptap 기반 블록 에디터 컴포넌트 구현
- `src/app/layout.tsx`: 전역 레이아웃 단순화 및 하위 레이아웃 위임

**주요 기술 (Key Tech Stack):**

- **Next.js Route Groups**: URL 경로에 영향을 주지 않으면서 프로젝트 파일 구조를 논리적으로 분리하는 데 사용
- **Tiptap**: Headless 방식의 텍스트 에디터로, 커스텀 블록 시스템 구축에 활용
- **Tailwind CSS**: 플랫폼과 데모 사이트 각각의 디자인 테마 적용

**핵심 기능 (Key Features):**

- **Multi-tenancy Simulation**: 단일 도메인 내에서 플랫폼 랜딩과 테넌트 데모 사이트의 공존 구현
- **Tenant Layout Separation**: 성격이 다른 두 영역에 대해 서로 다른 헤더/푸터 및 레이아웃 적용
- **Block Editor**: 관리자가 콘텐츠를 직접 생성하고 수정할 수 있는 위지윅 편집 기능

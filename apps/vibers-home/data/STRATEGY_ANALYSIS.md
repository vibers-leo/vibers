## PART A: 현재 상태 진단

### 계발자들은 무엇인가?

린캔버스와 워크북을 보면, 계발자들의 본질은 이것이다:

> **"소상공인의 비즈니스 인프라(홈페이지/앱)를 만들고 관리해주는 1인 기술 파트너"**

이것은 2-Track으로 구현된다:
- **Track A (단건 제작)**: 50만~300만원, 문 여는 용도
- **Track B (CTO 구독)**: 월 50~500만원, 장기 관계

그리고 팬이지에서 이미 검증된 핵심 인사이트가 있다:
- 고객은 "CTO"라는 단어를 이해 못 한다
- "구독"이라는 프레이밍에 거부감이 있다
- "유지관리비", "서버관리비"는 자연스럽게 수용한다
- 고객은 CTO 자문이 아니라 **"만들어달라 + 관리해달라"**를 원한다

### 현재 전략의 문제 2가지

#### 문제 1: 프레이밍과 고객 언어의 괴리

린캔버스에서 이미 스스로 지적했다:

> "CTO 구독은 우리 관점의 프레이밍이고, 고객 관점에서는 내 비즈니스를 위한 홈페이지/앱을 만들고 관리해줄 파트너를 찾고 있음"

이 발견이 있었는데도 서비스명이 여전히 "CTO 구독"이고, 티어명이 BASIC/PRO/MASTER다. 고객이 이 이름을 듣고 "아, 이게 나한테 필요한 거구나"라고 느끼는가? 아마 아니다.

#### 문제 2: 계발자들(Vibers)과 팬이지(FanEasy)의 관계가 불명확

- 팬이지가 이미 소상공인 홈페이지를 만들어서 매출을 내고 있다
- 계발자들은 같은 일을 다른 브랜드로 하려는 것인가?
- 아니면 계발자들이 상위 브랜드이고 팬이지는 하위 제품인가?
- 현재 린캔버스에서 "팬이지 접점"을 경쟁우위로 적었는데, 관계 정의가 안 되어 있다

## PART B: 브랜드 네이밍 전략 분석

### "계발자들" vs "개발자들" — 비판적 평가

멘토님이 "개발자들"을 제안했다고 했는데, 이건 단순한 이름 선택이 아니라 **서비스 정체성의 선택**이다.

#### 옵션별 분석

| 기준 | 계발자들 (啓發者들) | 개발자들 (開發者들) | Vibers |
|------|:---:|:---:|:---:|
| **1차 인식** | "계발? 자기계발?" 혼란 | "아, 개발자" 즉시 이해 | "바이브? 바이버스?" 외래어 |
| **타겟 연결** | 소상공인이 이해 못 할 수 있음 | 소상공인도 "개발자"는 안다 | 트렌디하지만 의미 불분명 |
| **검색 SEO** | "계발자" 검색량 거의 없음 | "개발자" 검색량 매우 높음 (경쟁도 높음) | "vibers" 고유 키워드 |
| **도메인** | vibers.co.kr (확보됨) | gaebaljadeul.kr? (미확인) | vibers.co.kr |
| **차별화** | 독특하지만 설명 필요 | 직관적이지만 범용적 | 고유하지만 한국 시장 약점 |
| **확장성** | "계발" = 성장/깨달음 → 교육/성장까지 확장 | "개발" = 코딩/제작 → 기술 서비스에 한정 | 글로벌 확장 가능 |

#### 깊은 분석: 이름이 줘야 하는 것

PM Skills의 Positioning 프레임워크로 보면, 브랜드명은 3가지를 만족해야 한다:
1. **즉시 이해** — 타겟이 이름만 듣고 무슨 서비스인지 감 잡아야 한다
2. **차별화** — 경쟁자와 구별되어야 한다
3. **약속** — 고객에게 어떤 가치를 줄지 암시해야 한다

| 기준 | 계발자들 | 개발자들 |
|------|---------|---------|
| 즉시 이해 | ❌ "계발자? 뭐하는 데지?" | ✅ "개발자들이 모여있나?" |
| 차별화 | ✅ 유일무이 | ❌ "개발자들"은 너무 일반적 |
| 약속 | 🟡 "계발 = 성장시켜준다" (설명 후) | 🟡 "개발 = 만들어준다" (즉시) |

#### 제3의 옵션 검토

"계발자들"도 "개발자들"도 완벽하지 않다면, 다른 방향도 고려해볼 수 있다:

| 옵션 | 장점 | 단점 |
|------|------|------|
| **계발자들 (유지)** | 도메인 확보됨, 독특함 | 1차 인식 어려움 |
| **개발자들 (변경)** | 직관적 | 범용적, 도메인 미확보 |
| **계발자들 + 서브타이틀** | 이름은 유지 + 설명 보완 | 조합이 길어질 수 있음 |
| **Vibers (영문 강조)** | 글로벌, 도메인 있음 | 한국 소상공인 타겟과 거리 |

#### 권고: "계발자들" 유지 + 서브타이틀 강화

**이유:**

1. **"개발자들"은 너무 흔하다** — 구글에 "개발자들" 검색하면 채용, 커뮤니티, 뉴스가 쏟아진다. 이 경쟁에서 소상공인 대상 웹 에이전시가 눈에 띌 수 없다.

2. **"계발자들"은 설명이 필요하지만, 설명이 곧 스토리가 된다** — "왜 계발인가요?" → "고객의 사업을 계발(啓發, 깨우쳐 발전시킴)하는 사람들이라는 뜻입니다" → 이것이 브랜드 스토리.

3. **도메인이 이미 있다** — vibers.co.kr은 확보되어 있고, "계발자들"이라는 한글과 "Vibers"라는 영문이 쌍으로 작동.

4. **확장성** — "개발"은 코딩에 한정되지만, "계발"은 교육, 성장, 컨설팅까지 확장 가능. CTO 구독 모델과도 더 잘 맞는다.

**단, 서브타이틀이 반드시 필요하다:**

| 현재 | 권장 |
|------|------|
| 계발자들 | 계발자들 |
| (설명 없음) | **"사장님의 기술 파트너"** |

또는:
- "계발자들 — 비즈니스 인프라를 만들어드립니다"
- "계발자들 — 사업을 성장시키는 기술 파트너"
- "계발자들 — 홈페이지부터 관리까지, 한 팀처럼"

이 서브타이틀이 있으면 "계발"이 뭔지 몰라도 서비스를 즉시 이해할 수 있다.

## PART C: 전략 개선안

### 1. Vibers와 FanEasy의 관계 정리

이것이 가장 먼저 명확해져야 한다. 세 가지 옵션:

| 옵션 | 구조 | 장점 | 단점 |
|------|------|------|------|
| A. 별개 브랜드 | Vibers = CTO 구독 / FanEasy = 인플루언서 에이전시 | 각자 타겟 명확 | LEO 1인이 2개 브랜드 운영 부담 |
| B. Vibers가 상위 | Vibers(모회사) → FanEasy(소상공인 사이트), NusuCheck 등 | 통합 브랜드 | Vibers 인지도가 아직 없음 |
| C. FanEasy 흡수 | Vibers가 FanEasy의 기능을 포함 | 리소스 집중 | FanEasy의 인플루언서 모델이 묻힘 |

**권고: 옵션 A (별개 브랜드, 역할 분리)**

이유:
- **Vibers(계발자들)** = LEO 개인의 기술 파트너 브랜드. "사장님, 저에게 맡기세요"
- **FanEasy(팬이지)** = 인플루언서 네트워크 기반 플랫폼. "인플루언서가 소개하는 웹 파트너"
- 타겟이 다르다: Vibers는 직접 영업/레퍼럴, FanEasy는 인플루언서 채널
- 하지만 **백엔드는 공유**: 같은 기술 스택, 같은 블록 시스템, 같은 인프라
- LEO가 Vibers로 직접 수주한 프로젝트는 Vibers 포트폴리오에, FanEasy 채널 프로젝트는 FanEasy에

```
LEO (1인)
  ├── Vibers (계발자들) — 직접 영업 브랜드
  │     "사장님의 기술 파트너"
  │     vibers.co.kr
  │     수익: 단건 제작 + CTO 관리비
  │
  └── FanEasy (팬이지) — 인플루언서 채널 브랜드
        "인플루언서가 소개하는 웹 파트너"
        faneasy.kr
        수익: 인플루언서 소개 → 제작비 + 관리비
```

### 2. Value Proposition 재설계

현재 UVP: "파트너의 마음으로, 비즈니스를 성장시키기 위한 인프라를 만들어드립니다"
→ 나쁘지 않지만 너무 추상적. 고객이 "그래서 뭘 해주는 건데?"라고 물을 수 있다.

**6-Part JTBD로 재구성:**

| 파트 | 내용 |
|------|------|
| Who | 홈페이지/앱이 필요한 소상공인 대표 (예산 100~500만원) |
| Why | 온라인 전환이 급한데, 누구한테 맡길지 모르겠고, 비용이 무섭다 |
| What Before | 크몽 검색 → 선택 불가. AI로 직접 → 불안. 아임웹 → 직접 해야 함. 대행사 → 홈페이지 안 만듦 |
| How | 26개 포트폴리오 확인 → 가격표 보고 선택 → 3일~4주 내 완성 → 커스텀 도메인/SEO/SNS 연동 |
| What After | 프로페셔널 사이트 보유, 검색 노출, 수정/관리 걱정 없음, 월 관리비로 기술팀 효과 |
| Alternatives | 크몽(품질 불안), 직접(시간 부족), 아임웹(한계), 안 함(매출 감소) |

**개선된 UVP 후보:**

| # | UVP | 강점 |
|---|-----|------|
| 1 | "26개 프로젝트로 증명된, 사장님만의 기술팀" | 포트폴리오 강조 + 팀 소속감 |
| 2 | "홈페이지 제작부터 관리까지, 월 관리비 하나로" | 명확한 서비스 범위 + 가격 암시 |
| 3 | "만들고 끝이 아닙니다. 사업이 성장할 때까지." | 장기 파트너십 약속 |

**권고: #2를 메인, #1을 서브로**
- 메인: "홈페이지 제작부터 관리까지, 월 관리비 하나로"
- 서브: "26개 프로젝트로 증명된 기술 파트너"

### 3. 가격 프레이밍 재설계 (팬이지 인사이트 반영)

현재 문제:
- "CTO 구독 BASIC/PRO/MASTER" → 고객이 CTO를 모르고, 구독에 거부감 있음
- 팬이지에서 "유지관리비"로 프레이밍했을 때 수용 검증됨

**개선안: 서비스 프레이밍을 고객 언어로**

#### Track A: 제작 (1회성)

| 현재 | 개선 | 이유 |
|------|------|------|
| 랜딩페이지 50만원~ | **홈페이지 개설** 80만원~ | "개설"이 소상공인에게 익숙 (사업자등록 "개설") |
| 비즈니스 웹사이트 100만원~ | **비즈니스 홈페이지** 150만원~ | 그대로 |
| 웹앱 200만원~ | **맞춤 서비스 제작** 200만원~ | "웹앱"은 소상공인이 모름 |

#### Track B: 관리 (월 정기) — "CTO 구독" → "기술 관리"로 재명명

| 현재 | 개선 | 월 비용 | 고객이 듣는 말 |
|------|------|--------|--------------|
| BASIC | **기본 관리** | 30~50만원 | "서버 돌아가게 해주고, 간단한 수정 해주시는 거죠" |
| PRO | **전담 관리** | 100~200만원 | "매달 만나서 사이트 개선해주시는 거죠" |
| MASTER | **기술 파트너** | 300~500만원 | "우리 회사 CTO처럼 기술 전체를 봐주시는 거죠" |

**핵심 변경:**
- "구독" → "관리" (구독 거부감 제거)
- "CTO" → 최상위 티어에만 사용 (알아듣는 고객만 이 티어를 선택)
- "BASIC/PRO/MASTER" → "기본/전담/파트너" (한국어로, 직관적으로)

### 4. Growth Loop — Vibers 전용

팬이지와 다른, Vibers만의 성장 루프:

```
LEO가 1개 사이트를 최선을 다해 만듦
    │
    │ ① 만족한 고객
    ▼
"관리도 해주세요" → 월 관리 계약
    │
    │ ② 포트폴리오에 추가
    ▼
vibers.co.kr 포트폴리오가 풍부해짐
    │
    │ ③ 포트폴리오를 본 새 고객이 문의
    │ ④ 만족한 고객이 지인 소개
    ▼
새 프로젝트 수주
    │
    │ (루프 반복)
    ▼
포트폴리오 30개 → 50개 → 차별화 압도적
```

**이 루프의 핵심 연료: 완성된 고객 사이트의 품질**

루프가 돌려면:
1. vibers.co.kr에 **포트폴리오 페이지** 필수 (현재 개발 중)
2. 각 프로젝트의 **before/after + 고객 후기** 필요
3. 가격표가 **명확히 공개**되어야 문의 허들 낮아짐
4. "문의 → 견적 → 계약" 플로우가 **3단계 이내**여야 함

### 5. 경쟁 포지셔닝 — Vibers의 고유 자리

```
           셀프서비스
              │
   아임웹 ─── │ ──── Wix
   카페24     │      Squarespace
              │
  범용 ───────┼──────── 전문
              │
   크몽       │     ★ Vibers
   위시켓     │     (포트폴리오 증명 +
              │      제작부터 관리까지)
              │
           풀서비스 대행
```

Vibers는 크몽(범용 프리랜서 마켓)과 에이전시(고가) 사이에 위치:
- 크몽보다 **신뢰도 높음** (포트폴리오 26개, 1인이 책임)
- 에이전시보다 **가격 합리적** (1인 운영이라 오버헤드 없음)
- 아임웹보다 **자유도 높음** (커스텀 개발 가능)

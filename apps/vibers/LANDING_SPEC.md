# Vibers (계발자들) — 랜딩 페이지 기획서

> 작성: Cowork (사업기획)
> 대상: AntiGravity (개발)
> 날짜: 2026-03-25
> 디자인 참고: 팬이지 CPR마케팅 페이지 (그리드, 레이아웃, 컴포넌트 스타일)

---

## 브랜드 정의

**계발자들 (Vibers)**
> "아이디어를 실현하는 기술 파트너"

- "계발(啟發)" = 깨우치다, 발전시키다 + "개발자"의 합성
- 포지셔닝: 에이전시/스튜디오 (클라이언트 대상)
- 타겟: 기술 개발이 필요한 소상공인, 프랜차이즈, 스타트업, 인플루언서
- 톤: 전문적이면서도 친근한, "우리가 만들어드립니다"

---

## 페이지 구조 (5섹션)

### 섹션 1: 히어로 (Hero)

**목적:** 첫 인상 — "여기가 뭐하는 곳인지" 3초 안에 전달

```
[히어로]
메인 카피: "계발자들과 함께 아이디어를 실현하세요."
서브 카피: "웹사이트, 앱, AI 서비스까지 — 기획부터 개발, 운영까지 함께합니다."
CTA 버튼: "프로젝트 문의하기" (→ 섹션5 문의로 스크롤)
보조 버튼: "포트폴리오 보기" (→ 섹션2로 스크롤)
```

**디자인:**
- 사이버펑크 배경 유지 (기존 DESIGN_GUIDE.md 준수)
- 히어로 배경에 미묘한 그리드 패턴 + 네온 글로우
- 통계 배지: "프로젝트 26+ / 서비스 중 10+ / 기술스택 15+"

---

### 섹션 2: 포트폴리오 (Portfolio / 사례)

**목적:** 실제 결과물을 보여줘서 신뢰 확보

**구조:**
```
[포트폴리오 그리드]
카드 형태로 최근 사례 표시:

카드 구성:
┌─────────────────────────┐
│  [스크린샷/썸네일]        │
│                         │
│  올루올루 홈페이지        │  ← 프로젝트명
│  by 팬이지              │  ← 작성자(브랜드)
│  #웹사이트 #소상공인      │  ← 태그
│  "3일 만에 완성"         │  ← 한줄 후기/설명
└─────────────────────────┘
```

**표시할 사례 (초기):**
| 사례명 | 작성자(브랜드) | 카테고리 |
|--------|-------------|---------|
| 올루올루 홈페이지 | 팬이지 (FanEasy) | 웹사이트 |
| 메이트체크 | 위로 (Wero) | 앱 서비스 |
| 프리미엄페이지 | 디랩 (D.Lab) | 전자카탈로그 |
| 누수체크 | 누수체크 | 서비스 사이트 |
| 러너스커넥트 | 디어스 (DUS) | 커뮤니티 |
| 가보자고 | 디어스 (DUS) | 모바일 초대장 |

**관리:**
- 관리자 페이지에서 사례 추가/수정/삭제/순서 변경 가능
- 각 카드 클릭 → 상세 페이지 또는 외부 URL로 이동
- "더 보기" 버튼 → /projects 아카이브 페이지

**디자인 참고:** CPR마케팅 페이지의 그리드 레이아웃
- 반응형: 모바일 1열 → 태블릿 2열 → 데스크탑 3열
- 카드 호버 시 살짝 확대 + 네온 보더 강조
- 썸네일은 16:10 또는 4:3 비율 통일

### 섹션 3: 서비스 소개 (What We Do)

**목적:** "우리가 뭘 할 수 있는지" 명확하게

```
[서비스 카드 그리드 — 2x2 또는 1x4]

┌──────────────┐  ┌──────────────┐
│  🌐 웹사이트  │  │  📱 앱 개발   │
│  기획~배포    │  │  iOS/Android  │
│  3일~2주     │  │  Expo/Native  │
└──────────────┘  └──────────────┘
┌──────────────┐  ┌──────────────┐
│  🤖 AI 서비스 │  │  🔧 기술 자문  │
│  챗봇, 자동화  │  │  CTO 구독     │
│  맞춤 AI     │  │  월 50만원~   │
└──────────────┘  └──────────────┘
```

---

### 섹션 4: 가격표 (Pricing) ⭐ 핵심 섹션

**목적:** 단건 의뢰부터 CTO 구독까지 한눈에

```
[가격 테이블]

───────────────────────────────────────────────
  단건 의뢰          │  CTO 구독 서비스
───────────────────────────────────────────────

  랜딩페이지    50만원~ │  BASIC     월 50만원
  웹사이트    100만원~ │  ├ 매주 계획서/보고서
  웹앱       200만원~ │  ├ 채팅/이메일 QA
  모바일앱    300만원~ │  └ 기술 방향 검토
  AI 서비스   협의     │
                     │  PRO      월 200만원
  * 규모/복잡도에 따라  │  ├ BASIC 전체 포함
    변동 가능         │  ├ 월 1회 원격 미팅
                     │  └ 개발 방향 설정 참여
                     │
                     │  MASTER   월 500만원
                     │  ├ PRO 전체 포함
                     │  ├ 주 1회 원격 미팅
                     │  ├ 월 1회 현장 방문
                     │  ├ 채용 면접 동석
                     │  └ 기술 실사 지원
───────────────────────────────────────────────

  [프로젝트 문의하기]   [구독 상담하기]
```

**디자인:**
- PRO 플랜에 "추천" 배지 (가장 많이 선택하는 플랜으로 유도)
- MASTER는 앵커 효과 — PRO가 합리적으로 보이게
- 단건 의뢰는 심플한 리스트, CTO 구독은 카드형 3열
- "추가 과업 시 별도 비용 발생" 하단 작게 표시

**중요:** 가격은 관리자 페이지에서 수정 가능하도록 DB화

### 섹션 5: 문의 (Contact)

**목적:** 전환 — 실제 문의 유입

```
[문의 폼]
이름: _______________
연락처: ______________
문의 유형: [단건 의뢰 / CTO 구독 / 기타]
프로젝트 설명: _________________
                 _________________
[문의 보내기]

또는: contact@dlab.co.kr / 카카오톡 채널
```

**처리:**
- 폼 제출 → Supabase에 저장 + 이메일 알림 (LEO에게)
- 카카오톡 채널 연동 (선택)

## 네비게이션

```
[로고: 계발자들]  포트폴리오  서비스  가격  문의  [바이버스 앱 →]
```

- 스크롤 시 각 섹션으로 스무스 스크롤
- 모바일: 햄버거 메뉴
- "바이버스 앱 →"은 /app 페이지로 (기존 유지)

---

## 기존 페이지와의 관계

| 기존 | 변경 후 |
|------|--------|
| `/` 히어로 + 프로젝트 카드 | → 5섹션 랜딩 (히어로 + 포트폴리오 + 서비스 + 가격 + 문의) |
| `/projects` 아카이브 | → 유지 (포트폴리오 "더 보기"에서 연결) |
| `/projects/[slug]` 상세 | → 유지 |
| `/app` 바이버스 앱 | → 유지 |
| `/admin` | → 신규 (관리자 페이지, ADMIN_SPEC.md 참고) |

## 디자인 가이드라인

- **기존 DESIGN_GUIDE.md 100% 준수** (사이버펑크, 네온그린, 다크모드)
- **CPR마케팅 페이지 참고**: 그리드 시스템, 카드 레이아웃, 섹션 간 리듬감
- 각 섹션 진입 시 Framer Motion 페이드인 + 슬라이드업
- 포트폴리오 카드: 호버 시 스케일 + 네온 글로우
- 가격표: PRO에 네온 보더 강조

---

## 개발 우선순위

1. **Phase 1**: 히어로 + 포트폴리오 (하드코딩 데이터) + 문의 폼
2. **Phase 2**: 가격표 + 서비스 소개
3. **Phase 3**: 관리자 페이지 (Supabase 연동) → 포트폴리오 CRUD
4. **Phase 4**: 문의 폼 → Supabase 저장 + 이메일 알림

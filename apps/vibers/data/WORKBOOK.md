## 2. Why — 왜 이 사업을 하는가

### 핵심 동기

> "내가 가진 능력으로 돈을 벌 수 있음을 증명하고 싶다."

이것은 단순히 "개발을 해서 돈을 벌겠다"가 아니라, **자신의 기술과 경험이 시장에서 실제로 가치 있음을 입증**하는 과정이다.

### 이것이 사업 방향에 주는 영향

- "증명"이 목표이므로 → **빠른 결과**가 중요하다. 1년 준비 후 런칭이 아닌, 바로 매출이 발생하는 구조
- 능력으로 버는 것이므로 → **서비스업(제작/자문)**이 자연스럽다. 제품을 만들어 스케일하는 건 다음 단계
- 가진 것(26개 포트폴리오)이 이미 충분하므로 → **새로 만들기보다 있는 것을 팔기**

## 3. 시장 검증 — 고객 인터뷰

### 인터뷰 원칙

- **팔지 않고, 듣기만 한다** — "이거 사시겠어요?"가 아닌 "어떻게 하고 계세요?"
- 최소 3건, 이상적으로 10건
- 대면 또는 통화, 서면 보조

### 인터뷰 결과 (3건 완료)

#### 인터뷰 1: 김상혁 (창업초기 대표)

| 항목 | 내용 |
|------|------|
| 세그먼트 | 소상공인 (창업 초기) |
| 핵심 니즈 | 홈페이지/앱 개발 |
| 예산 감각 | 100~500만원이지만 **망설이는 중** |
| 현재 대안 | 없음 (아직 시작 못함) |
| 핵심 인사이트 | **저렴한 개설비 + 낮은 유지관리비 허들**이 진입 키 |
| 예상 전환 경로 | 단건 제작(50~100만원) → 만족 시 BASIC 구독 전환 |

#### 인터뷰 2: 정진우 (프랜차이즈 운영)

| 항목 | 내용 |
|------|------|
| 세그먼트 | 프랜차이즈 |
| 핵심 니즈 | 홈페이지가 필요한지조차 모름 |
| 예산 감각 | 마케팅에 월 30~50만원 이미 지출 중 |
| 현재 대안 | 마케팅 대행사 (네이버/키워드 광고) |
| 핵심 인사이트 | **기존 마케팅 대행사와 연계** 가능, 홈페이지+관리+마케팅 번들 제안이 효과적 |
| 예상 전환 경로 | "마케팅비에 홈페이지 관리 포함" 프레이밍 → BASIC 월 50만원 부담 없음 |

#### 인터뷰 3: 강규원 (프랜차이즈 본사)

| 항목 | 내용 |
|------|------|
| 세그먼트 | 프랜차이즈 본사 |
| 핵심 니즈 | 가맹 사업 확장용 인프라 |
| 예산 감각 | 미확인 (규모 있을 것으로 추정) |
| 현재 대안 | 미확인 |
| 핵심 인사이트 | **프랜차이즈 전문 홈페이지 + 빠른 대응 + 전체 인프라** 원함 |
| 예상 전환 경로 | 단건 제작(200~300만원) → PRO/MASTER 전환 가능성 높음 |

### 인터뷰에서 도출한 핵심 발견

> **고객은 "CTO 자문"이 아니라 "홈페이지/앱을 만들어달라"를 원한다.**
>
> "CTO 구독"은 우리 관점의 프레이밍이고, 고객 관점에서는 **"내 비즈니스를 위한 홈페이지/앱을 만들고 관리해줄 파트너"**를 찾고 있다.

### 인터뷰 결과에 따른 전략 수정

| 항목 | 변경 전 | 변경 후 |
|------|--------|--------|
| 서비스 프레이밍 | "CTO를 빌려드립니다" | "비즈니스 인프라(홈페이지/앱) 제작 파트너" |
| 진입 전략 | CTO 구독 직접 세일즈 | 단건 제작으로 진입 → 신뢰 후 구독 업셀 |
| 1차 타겟 | 소상공인 + 프랜차이즈 + 스타트업 | **소상공인 Only** (프랜차이즈는 2차) |
| 스타트업 | 포함 | **제외** (니즈 다르고 세일즈 사이클 김) |

### 추가 인터뷰 계획

- [ ] 기존 고객(팬이지, 누수체크 등) 5~10명에게 안부 + 새 서비스 소개
- [ ] 소상공인 커뮤니티/교육기관 접촉 → 인터뷰 대상 확보
- [ ] 1인 마케터 2~3명 대상 파트너십 인터뷰

## 4. 경쟁/대체재 분석

### 고객이 지금 쓰고 있는 대안들

| 대안 | 고객이 쓰는 이유 | 치명적 불만 | 우리 공략 포인트 |
|------|---------------|-----------|---------------|
| **크몽/위시켓** | 저렴한 개발 | 너무 많은 선택지, 품질 편차 | 포트폴리오 중심 + 가격표와 결과물 예시 제공 |
| **생성형 AI로 직접** | 원하는 대로 가능 | 완성·보안·성공 확신 없음 | 전문지식과 경험 + 빠른 보완 |
| **아임웹/워드프레스** | 비용 저렴 | 직접 하거나 외주 필요 | 이런 도구를 능숙하게 다루는 전문성 |
| **마케팅 대행사** | 네이버/키워드 광고 | 홈페이지 자체는 안 만들어줌 | 홈페이지+관리+마케팅 번들 가능 |
| **아무것도 안 함** | 귀찮음/무서움/모름 | 매출 감소 지속 | "다 해드립니다" — 최소한의 의사결정만 요구 |

### 빈틈 공략 핵심

> 크몽은 "누구한테 맡길지" 문제를 해결 못하고,
> AI는 "완성/보안" 문제를 해결 못하고,
> 아임웹은 "직접 해야 한다"는 문제를 해결 못하고,
> 마케팅 대행사는 "홈페이지 자체"를 안 만든다.
>
> → **"믿을 수 있는 파트너가 처음부터 끝까지 다 해주는 것"** 이 빈틈이다.

### 핵심 가치 제안 (UVP)

**내부용 (전략):**
> "파트너의 마음으로, 비즈니스를 성장시키기 위한 인프라(홈페이지, 앱)를 만들어드립니다."

**마케팅 헤드라인 (고객 대면):**
> "빠르게 비즈니스를 시작하는 홈페이지"

**왜 우리에게 맡겨야 하는가:**
- 실제 프랜차이즈/소상공인 홈페이지 포트폴리오가 있다 (증거)
- 만들고 끝이 아니라, 관리까지 해준다 (안심)
- 가격표와 결과물 예시가 명확하다 (투명)
- AI+바이브코딩으로 빠르고 저렴하다 (효율)

### BEP (손익분기점)

> **월 매출 260만원** 달성 시 BEP
> 최소 시나리오: 단건 1건(150만원) + BASIC 2곳(100만원) = **250만원** ≈ BEP

### 매출 목표 시나리오

## 11. 체계화 — 반복 가능한 프로세스

> 이 과정을 템플릿화하여, **새 사업 아이디어가 떠오를 때마다** 동일하게 적용

### 9-Step 사업개발 프로세스

| Step | 내용 | 산출물 | 소요 시간 |
|------|------|--------|---------|
| 1 | **한 줄 정의** | "[타겟]을 위한 [솔루션] — [핵심 가치]" | 30분 |
| 2 | **출발선 점검** | 런웨이 계산, Why 선언, 3개월 목표 | 1시간 |
| 3 | **고객 인터뷰 3건** | 팩트 시트 (팔지 말고 듣기만) | 1~2주 |
| 4 | **경쟁/대체재 분석** | 빈틈 공략 포인트 도출 | 2~3시간 |
| 5 | **UVP 도출** | 고객 언어로 된 한 줄 가치 제안 | 1시간 |
| 6 | **린캔버스 9칸** | 비판적 검토(⚠️) 포함 | 2~3시간 |
| 7 | **MVP 실험 설계** | Fake Door Test (전환율 3% 기준) | 1시간 |
| 8 | **90일 GTM 플랜** | 채널 3곳 + 첫 10명 획득법 | 2시간 |
| 9 | **개발 착수** | AntiGravity/Terminal Claude에 기획서 전달 | ~ |

### 각 단계 질문 체크리스트

**Step 1 — 한 줄 정의**
- 타겟이 누구인가? (직업, 상황, 규모)
- 무엇을 해결해주는가? (동사로 표현)
- 왜 우리가 해야 하는가? (경쟁 우위)

**Step 2 — 출발선 점검**
- 런웨이가 몇 개월인가?
- 3개월 안에 무엇을 달성해야 하는가?
- 주당 몇 시간을 투입할 수 있는가?

**Step 3 — 고객 인터뷰**
- 이 문제를 지금 어떻게 해결하고 계세요?
- 가장 힘든 점은 무엇인가요?
- 이 해결책에 얼마까지 쓸 수 있나요?
- (절대 팔지 않는다, 듣기만 한다)

**Step 4 — 경쟁 분석**
- 고객이 "지금" 쓰고 있는 대안은?
- 그 대안의 치명적 불만은?
- 그 불만을 우리가 해결할 수 있는가?

**Step 5 — UVP**
- 고객이 실제로 쓴 단어로 표현할 수 있는가?
- 3초 안에 이해되는가?
- 경쟁사와 다른가?

**Step 6~9 — 린캔버스~개발 착수**
- 이 워크북의 구조를 그대로 따른다

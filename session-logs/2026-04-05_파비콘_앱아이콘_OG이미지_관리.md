# 파비콘, 앱아이콘, OG 이미지 어드민 관리 기능

- 날짜: 2026-04-05
- 에이전트: [TCC]

## 세션 개요
vibers-home 어드민에 파비콘/앱아이콘/OG 이미지를 관리할 수 있는 UI를 구현하고,
기존 아이콘 미세팅 앱들에 로고로부터 자동 생성된 아이콘을 적용했다.

## 주요 논의

### IconManager 컴포넌트 (vibers-home/components/admin/IconManager.tsx)
- SEO 어드민 페이지(`/admin/seo`) 하단에 통합
- 개별 아이콘 업로드: favicon(64px), apple-touch-icon(180px), android(512px), OG(1200×630)
- **자동생성 패널**: 로고 1장 업로드 → favicon/apple-touch-icon/android 3종 자동 생성
- **OG 자동캡처**: Microlink API로 사이트 히어로 스크린샷 → 1200×630 리사이즈 → NCP 업로드
- 이미지 리사이즈: 브라우저 Canvas API (center-crop), File 또는 URL 입력 지원
- 업로드: `@vibers/storage/client`의 `uploadFile` (NCP Object Storage)
- Firebase Storage 사용 금지 원칙 준수

### 스토리지 설정
- vibers-home `package.json`에 `"@vibers/storage": "*"` 추가
- `app/api/storage/presign/route.ts` 추가 (패키지 템플릿에서 복사)
- `app/api/admin/icons/route.ts` 추가 (in-memory GET/POST)
- `packages/storage/src/types.ts`에 `'vibers-home'` StoragePlatform 추가

### 아이콘 미세팅 앱 일괄 처리
`sips` 명령어로 기존 로고를 자동 리사이즈하여 임시 아이콘 생성:
- art-way: `public/logo.png` → favicon.png(64), apple-touch-icon.png(180), icon-512.png(512)
- arthyun: `public/logo.png` → 동일 3종
- goodzz: `public/logo.png` → 동일 3종 (JPEG임에도 .png 확장자였음, sips 경고 발생하나 정상 처리)
- dlab: `public/logo.png` → apple-touch-icon.png(180), icon-512.png(512) (favicon.png 기존 존재)
- myratingis: 이미 proper metadata.icons 설정 완료 → 건드리지 않음

각 앱 `layout.tsx`에 `metadata.icons` 추가:
```ts
icons: {
  icon: [{ url: "/favicon.png", type: "image/png" }],
  apple: [{ url: "/apple-touch-icon.png" }],
},
```
